ACCORDION COMPONENT — SPECIFICATION

Extracted from Figma node 0:1 (❖ Accordion)
File: Telehealth App — Components

1. OVERVIEW
-----------
The Accordion component organizes content into collapsible sections. Users click
a trigger row (heading + chevron icon) to expand or collapse a content panel
beneath it. Ideal for FAQs, settings panels, and progressive disclosure.

2. ANATOMY
----------
  1. Container — Outer div with a bottom border. Houses trigger + content panel.
  2. Heading  — Text label in the trigger row describing the section.
  3. Icon     — Chevron-down when collapsed, rotates 180° (chevron-up) when expanded.
  4. Slot     — Content panel area revealed below the trigger when opened.

3. PROPERTIES (from Figma)
--------------------------
  | Property | Values              | Default   |
  |----------|---------------------|-----------|
  | size     | sm, md              | sm        |
  | type     | closed, opened      | closed    |

8. COMPONENT API
----------------

  AccordionItem (single item):
    | Prop          | Type                      | Default   | Description                                  |
    |---------------|---------------------------|-----------|----------------------------------------------|
    | heading       | string                    | required  | Text shown in the trigger row                |
    | size          | "sm" | "md"               | "sm"      | Size variant                                 |
    | disabled      | boolean                   | false     | Disables interaction                         |
    | open          | boolean                   | undefined | Controlled open state                        |
    | defaultOpen   | boolean                   | false     | Initial open state (uncontrolled)            |
    | onOpenChange  | (open: boolean) => void   | undefined | Callback on toggle                           |
    | children      | ReactNode                 | —         | Content for the expandable panel             |
    | className     | string                    | undefined | Extra CSS classes                            |

  Accordion (group wrapper):
    | Prop      | Type                    | Default  | Description                         |
    |-----------|-------------------------|----------|-------------------------------------|
    | type      | "single" | "multiple"   | "single" | Allow one or many open at once      |
    | size      | "sm" | "md"             | "sm"     | Default size for all children       |
    | disabled  | boolean                 | false    | Disable all children                |
    | children  | ReactNode               | —        | AccordionItem elements              |
    | className | string                  | undefined| Extra CSS classes                   |

9. ACCESSIBILITY
----------------
  - Trigger: native <button> with aria-expanded, aria-controls
  - Panel: role="region" with aria-labelledby pointing to the trigger
  - Disabled: native disabled attribute on the button
  - Keyboard: Tab to focus, Enter/Space to toggle
  - Focus ring: focus-visible:shadow-focus (avoids showing on click)

10. USAGE GUIDELINES
--------------------
  DO:
    ✓ Use for FAQs, settings, and progressive disclosure
    ✓ Keep headings short and descriptive
    ✓ Choose single vs multiple mode based on content
    ✓ Provide clear expanded/collapsed visual feedback

  DON'T:
    ✗ Hide critical information in accordions
    ✗ Nest accordions inside accordions
    ✗ Use vague headings like "Section 1" or "More info"
    ✗ Mix sizes within the same accordion group
