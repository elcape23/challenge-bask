1. COMPONENT OVERVIEW
─────────────────────

Counter lets users adjust a numeric value incrementally with
decrement (−) and increment (+) buttons. It combines both buttons
and a value display in a bordered, compact layout.

Component type: Form input / Stepper
Interaction: Click buttons to step value up or down

────────────────────────────────────────

2. ANATOMY
──────────

  ┌───┬──────┬───┐
  │ − │  3   │ + │
  └───┴──────┴───┘

  1. Container — bordered flex row, rounded corners
  2. Decrement button — minus icon, left position
  3. Value display — centered text showing current number
  4. Increment button — plus icon, right position

5. STATES
─────────

  DEFAULT
    All buttons interactive
    Value within min/max range

  HOVER (button)
    Background: var(--color-background-surface-neutral-hover) = #d3d6cc

  ACTIVE (button)
    Background: var(--color-background-fill-neutral-default) = #e4e6de

  MIN REACHED
    Decrement button: disabled (opacity 50%, cursor not-allowed)
    Increment button: still active

  MAX REACHED
    Increment button: disabled (opacity 50%, cursor not-allowed)
    Decrement button: still active

  DISABLED (whole component)
    Opacity: 50% on the entire container
    Cursor: not-allowed
    Both buttons disabled

8. ACCESSIBILITY
────────────────

  - Container: role="group" aria-label="Counter"
  - Decrement: aria-label="Decrease"
  - Increment: aria-label="Increase"
  - Value: aria-valuenow, aria-valuemin, aria-valuemax, aria-live="polite"
  - Disabled buttons use native disabled attribute
  - Keyboard: Tab between buttons, Enter/Space to activate
  - Focus ring: shadow-focus (3px)

────────────────────────────────────────

9. CONTROLLED / UNCONTROLLED SUPPORT
─────────────────────────────────────

  Controlled mode:
    - Pass `value` prop — component reflects external state
    - Must provide `onChange` handler to update state

  Uncontrolled mode:
    - Pass `defaultValue` prop for initial state
    - Internal state tracks changes via useState

  The component determines mode from `value !== undefined`.
  Values are always clamped to [min, max] on each update.
