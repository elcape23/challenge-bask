1. COMPONENT OVERVIEW
─────────────────────

Cards are flexible container components that group a heading and a slot
(children) into a visually distinct surface. They support three sizes,
three background treatments, and a toggleable heading.

Component type: Container / Layout
Interaction: Non-interactive (container only)

────────────────────────────────────────

2. ANATOMY
──────────

  ┌───────────────────────────┐
  │  ┌─────────────────────┐  │
  │  │  1. .heading         │  │  (optional sub-component)
  │  └─────────────────────┘  │
  │  ┌─────────────────────┐  │
  │  │                     │  │
  │  │  2. Slot (children) │  │  (flexible content area)
  │  │                     │  │
  │  └─────────────────────┘  │
  └───────────────────────────┘

  1. .heading — optional text heading, typography scales with card size
  2. Slot — main content area that fills remaining space (flex: 1)

────────────────────────────────────────

3. PROPERTIES
─────────────

| Prop         | Type                                 | Default    | Description                                      |
|------------- |--------------------------------------|------------|--------------------------------------------------|
| size         | "lg" | "md" | "sm"                   | "lg"       | Size variant — padding, gap, radius, heading     |
| background   | "solid" | "blur" | "transparent"     | "solid"    | Background treatment                             |
| showHeading  | boolean                              | true       | Toggle the heading sub-component                 |
| heading      | string                               | "Heading"  | Text in the heading sub-component                |
| children     | ReactNode                            | undefined  | Slot content                                     |
| className    | string                               | undefined  | Additional CSS classes                            |

────────────────────────────────────────

8. SUB-COMPONENT: .heading
──────────────────────────

  A sub-component that renders the heading text within the card.
  Scales typography and padding based on the parent card's size.

  | Size | Typography          | Padding         |
  |------|---------------------|-----------------|
  | lg   | Heading/06/Medium   | 8px (space-2)   |
  | md   | Body/01/Medium      | 4px (space-1)   |
  | sm   | Body/02/Medium      | 2px (space-0)   |

────────────────────────────────────────

9. ACCESSIBILITY
────────────────

  - Cards are non-interactive containers — no role needed by default
  - Use semantic HTML inside the slot (headings, paragraphs, lists)
  - If made clickable, wrap with <a> or <button> and provide aria-label
  - Interactive children must be reachable via keyboard in logical order
