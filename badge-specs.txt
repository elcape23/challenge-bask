1. OVERVIEW
-----------
The Badge component displays compact, non-interactive status indicators,
categories, or semantic labels. Badges attach to other elements (list items,
cards, table rows) and communicate information at a glance.

2. ANATOMY
----------
  1. Container — Pill-shaped element (border-radius: 9999px) with a type-specific fill.
  2. Icon      — Optional leading icon slot (24px md / 20px sm). Type-specific icons: circle-info (neutral/info), check-circle (success), alert-triangle (warning), x-circle (danger).
  3. Label     — Short text label using medium-weight typography.

  | size     | sm, md                                             | md        |

---------

7. MODES

  Dark mode inverts token references (not yet implemented).

9. ACCESSIBILITY
----------------
  - Non-interactive: badges do not receive focus or click handlers
  - Icons use aria-hidden="true" (decorative only)
  - Don't rely on color alone — label text must be descriptive
  - Maintain sufficient contrast between label and background

10. USAGE GUIDELINES
--------------------
  DO:
    ✓ Use for short, glanceable status indicators
    ✓ Be consistent with type-to-meaning mapping
    ✓ Keep labels to one or two words
    ✓ Use neutral for non-semantic labels

  DON'T:
    ✗ Use badges for long text or sentences
    ✗ Badge every item in a list
    ✗ Use decorative badges with no information value
    ✗ Mix semantic colors inconsistently
