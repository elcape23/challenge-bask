ALERT COMPONENT — SPECIFICATION
=================================
Extracted from Figma node 2:5 (❖ Alert)
File: Telehealth App — Components

1. OVERVIEW
-----------
The Alert component communicates status, feedback, or important information
through contextual, color-coded inline banners. Alerts persist until dismissed
or the triggering condition resolves.

2. ANATOMY
----------
  1. Container   — Rounded bordered surface with type-specific background.
  2. Icon        — Leading status icon (24px md, 20px sm).
  3. Heading     — Bold heading text summarizing the alert.
  4. Description — Supporting text with details or instructions.
  5. Button      — Optional underlined link-style action button.

6. LAYOUT
---------
  Horizontal flex row: [Icon] [Text Column: Heading + Description] [Button]
  - All elements vertically centered (items-center)
  - Text stacks vertically (heading on top, description below)
  - Button aligned to the right edge
  - Full width responsive (w-full in implementation)

7. MODES
--------
  Semantic modes: Light and Dark.
  Light mode uses the token set documented above.

9. ACCESSIBILITY
----------------
  - Container: role="alert" for assistive technology announcements
  - Icons: aria-hidden="true" (decorative; meaning conveyed by text)
  - Don't rely on color alone — pair with icon and text
  - Sufficient contrast ratios for all type/background combos

10. USAGE GUIDELINES
--------------------
  DO:
    ✓ Match type to message severity
    ✓ Keep heading short and actionable
    ✓ Place alerts near relevant content
    ✓ Use one action per alert

  DON'T:
    ✗ Use danger for non-critical information
    ✗ Stack multiple alerts — consolidate
    ✗ Auto-dismiss alerts (use Toast instead)
    ✗ Rely on color alone
